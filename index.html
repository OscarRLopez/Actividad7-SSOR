Active Directory

Active Directory es un servicio de directorio de Microsoft que corre en Windows Server y permite a los administradores administrar los permisos y accesar a los recursos de la red. Este servicio almacena a los datos como objetos y estos objetos por sí solos, pueden ser un usuario, un grupo, aplicación, dispositivo, entre otros elementos. Los elementos que definimos como un recurso dentro de la red son por ejemplo las impresoras o computadoras y por otro lado, los elementos que definimos como principios de seguridad, son los usuarios o grupos.

Una de las ventajas de los Active Directory es separar la estructura lógica de la organización (dominios) de la estructura física (topología de red).

  • Ordenadores
  • Servidores
  • Laptops
  • Cámaras de seguridad
  • Impresoras
  • Celulas
Y todos estos componentes conectados están registrados en una base de datos central ubicada en el controlador de dominio.

Herramientas para administración de un Active Directory

Las herramientas para la administración de un Directorio Activo generalmente se usan para:

  • Administración de Equipos de Active Directory
  • Administración de Permisos de Archivos de Active Directory
  • Administración de Usuarios de Active Directory
  • Administración de Contraseñas de Active Directory
  • Gestión de Contactos de Active Directory
  • Gestión de Grupos de Active Directory
Las herramientas que se utilizan son:
  • Editor ADSI (Active Directory Service Interfaces).
  • Dominios y confianzas de Active Directory.
  • Usuarios y equipos de Active Directory.
  • Sitios y servicios de Active Directory.
  • Centro de Administración de Active Directory.
  • Módulo de Active Directory para PowerShell.

Editor ADSI

ADSI se refiere a "Active Directory Service Interfaces" y estos son un conjunto de interfaces COM que se utilizan para acceder a las características de diferentes servicios de directorio incluso de diferentes proveedores de red.

Esta interfaz se utiliza en un ambiente de computación distributiva para presentar un único servicio de interfaz de directorio en una administración de recursos en la red. Este por lo tanto permite a los administradores y desarrolladores realizar tareas administrativas como agregar usuarios, administrar impresoras o localizar recursos en una red y también se pueden usar estos servicios para enumerar los recursos de un servicio de directorio.

El Editor ADSI es un complemento de Microsoft Management Console (MMC) que se usa para administar estos servicios de directorio ligero de Active Directory (AD LDS). Este administra una instancia de AD LDS, pero para esto se debe conectar y enlazar a distancia, lo cual permite que se puedan administrar objetos y contenedores de la instancia.

Dominios y confianzas de Active Directory

Estos son el complemento Microsoft Management Console (MMC) que puede ser utilizado para administrar confianzas de dominios, niveles funcionales del bosque o del dominio y sufijos de nombre principal de usuario (UPN). Una relación de confianza en el entorno Windows significa que un controlador de dominio confía en otro, permitiendo que se puedan relacionar y tener usuarios que puedan iniciar sesión en ambos controladores de dominio. Además se pueden compartir recursos que serán accesibles desde ambos extremos. Se tienen dos tipos de relaciones de confianza:

  • Bidireccionales: Ambos dominios confían mutuamente y no existen restricciones a la hora de hacer operaciones.
  • Unidireccionales: Un controlador de dominio confía en otro y le permite el acceso a sus usuarios y recursos, pero no sucede lo mismo del otro lado.

Usuarios y equipos de Active Directory

También es un complemento de Microsoft Management Console (MMC) que se puede usar para administrar y publicar información en el directorio. Con esto se puede:

  • Crear una cuenta de controlador de dominio de solo lectura.
  • Crear y administrar equipos, contactos, grupos, unidades organizativas, impresoras, usuarios, etc.
  • Delegar el control de tareas comunes sobre objetos de Active Directory en usuarios o grupos especificados.
  • Transferir maestros de operaciones a nivel de dominios.

Sitios y Servicios de Active Directory

De igual forma es un complemento de Microsoft Management Console (MMC) de Windows Server que tiene como función la administración de la replicación de datos de directorio entre todos los sitios de un bosque de los Servicios de Dominio de Active Directory (AD DS). Lo que se puede administrar con esta herramienta es:

  • Administración de sitios
    • Sitios
    • Subredes
    • Servidores
    • Configuración NTDS
    • Conexiones
    • Vínculos a sitios
    • Transportes entre sitios IP y SMTP
  • Publicación de servicios
  • Referencias adicionales

Centro de administración de Active Directory

En Windows Server se puede administrar y publicar información en los entornos de Active Directory mediante el complemento a usuarios y equipos del MMC y también es posible administrar los objetos del servicio de directorio a través del Centro de Administración de Active Directory. El Centro de Administración de AD brinda una mejor experiencia a través de la interfaz de línea de comandos PowerShell en cuanto a la administración de datos para poder realizar las tareas administrativas de los objetos mediante la navegación controlada por datos y la navegación orientada por tareas. Este se puede utilizar para las siguientes tareas:

  • Crear grupos, cuentas de equipo, contenedores, unidades organizativas o cuentas de usuario y administrar las ya existentes.
  • Conectar uno o varios dominios o controladores de dominio en la misma intancia del Centro de Admonostración de Active Directory.
  • Ver o administrar la información de directorio de los controladores de dominio.
  • Filtrar datos de Active Directory mediante la generación de consultas.

Módulo de Active Directory para PowerShell

Este módulo consolida a un grupo de cmdlets. Estos nos ayudan a administrar los dominios de Active Directory, los conjuntos de configuración de AD LDS y las instancias de la herramienta de montaje de bases de datos en un sólo paquete. Esta línea de comandos nos ayuda a controlar la administración de una manera más sencilla y productiva.

Conclusión personal

Honestamente yo no conocía sobre Active Directory antes de realizar la investigación para esta actividad, pero por lo que veo es bastante completo e importante en el ámbito de la administración de redes. Es muy útil el hecho de que con él puedes administrar las diferentes partes de una red, como permisos, recursos, usuarios, etc. Una de las cosas que normalmente no tomamos en cuenta al aprender sobre administración de servicios es lo complicado que esto se vuelve cuando se escala un servicio, una red compleja y grande no será igual de fácil de administrar que una pequeña red de prueba que realizamos para nuestras actividades escolares, y el tener estas poderosas herramientas ayudan mucho en el mundo real.
